Usage

Install the adapter and wrap your images with the img-fit component.

Install

npm install @danilo.riedel/img-fit-next

Component

import { ImgFitImage } from '@danilo.riedel/img-fit-next';

export default function Page() {
  return (
    <div style={{ position: 'relative', aspectRatio: '16 / 9' }}>
      <ImgFitImage
        src="https://cdn.example.com/photo.jpg"
        alt="Photo"
        params="f=webp&q=80"
        fill
      />
    </div>
  );
}

Configuration

Pass an options prop to control DPR, debounce, and more. The parent element must always have explicit dimensions when using fill.

Disable retina

<ImgFitImage
  src="https://cdn.example.com/photo.jpg"
  alt="Photo"
  options={{ dpr: false }}
  fill
/>

Request CSS pixel widths only — useful for thumbnails or low-bandwidth contexts.

Extra CDN parameters

<ImgFitImage
  src="https://cdn.example.com/photo.jpg"
  alt="Photo"
  params="f=webp&q=80"
  fill
/>
{/* Result: ?f=webp&q=80&rs=1250 */}

The params string is appended to the URL before ?rs=.

Parent dimensions

<div style={{
  position: 'relative',
  height: '400px'
}}>
  <ImgFitImage
    src="https://cdn.example.com/photo.jpg"
    alt="Photo"
    fill
  />
</div>

The fill prop requires the parent to have position: relative and explicit dimensions.

How it works

  • The component wraps next/image using the fill prop.
  • On the client it measures the parent container and builds the optimal URL.
  • Width is multiplied by devicePixelRatio (DPR) before snapping to cache-friendly breakpoints.
  • It appends ?rs=<width> and passes the result to next/image.
  • SSR stays stable — the URL is updated after hydration via useEffect.
