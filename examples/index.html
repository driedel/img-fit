Why img-fit?

Stop guessing breakpoints. Start serving the right image for every device and container.

📐

Zero breakpoints

The browser measures the real rendered container width at runtime. No guessing, no hard-coded sizes, no media queries in your JavaScript.

✨

Retina automatic

Multiplies by devicePixelRatio before snapping, so MacBook Retina, iPhone, and AMOLED screens all get sharp images without a single extra line of code.

⚡

Zero dependencies

The vanilla core is under 4 KB. Framework adapters are thin wrappers around the same core — no bundler plugins, no server config.

How Sizing Works

img-fit measures the rendered container width and snaps it up to the next cache-friendly breakpoint. Exact widths would fragment CDN caches, so every value is rounded using the following grid:

  • < 300 px → rounded to the next 20 px
  • 300 px – 1000 px → rounded to the next 50 px
  • > 1000 px → rounded to the next 250 px

Retina & High-DPR Screens

Devices like MacBook Retina, iPhone, and AMOLED phones have a devicePixelRatio greater than 1 — they fit 2 or 3 physical pixels into each CSS pixel. img-fit multiplies the container width by that ratio before snapping, so every screen gets a sharp image without extra config.

Your screen's pixel density: DPR = … via window.devicePixelRatio
Container (CSS px) DPR Raw width After snap ?rs= generated
560 px 1× 560 600 ?rs=600
560 px 2× 1 120 1 250 ?rs=1250
260 px 2× 520 550 ?rs=550
260 px 3× 780 800 ?rs=800

To request CSS pixel widths only (no DPR multiplication):

ImgFit.init({ dpr: false });

Quick Start

Install the package for your framework and add one data attribute.

Install

npm install @danilo.riedel/img-fit

Initialize

import ImgFit from '@danilo.riedel/img-fit';

ImgFit.init();
